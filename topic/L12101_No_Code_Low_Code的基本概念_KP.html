#1
★★★★★
No-Code (無程式碼) 平台的定義
核心概念
No-Code 平台是一種允許使用者在完全不需要編寫任何程式碼的情況下，透過視覺化介面（如拖放 - Drag-and-Drop）和預先構建的模組來建立應用程式的開發工具或環境。(課程規劃書 L121)
#2
★★★★★
Low-Code (低程式碼) 平台的定義
核心概念
Low-Code 平台同樣強調視覺化開發和預構建模組，但允許開發者在必要時編寫少量程式碼，以實現更高度的客製化、整合複雜系統或處理特定業務邏輯。它在易用性和彈性之間取得平衡。(課程規劃書 L121)
#3
★★★★★
No-Code 與 Low-Code 的核心差異
核心概念
最主要的差異在於是否需要以及允許多少程度的程式碼編寫。
  • No-Code：完全不需要寫程式碼，主要面向非技術背景的公民開發者。
  • Low-Code：主要透過視覺化開發，但允許少量程式碼以增加彈性與客製化能力，目標使用者包含公民開發者與專業開發者。
(公部門手冊 P80; 課程規劃書 L121)
#4
★★★★
No-Code/Low-Code 的核心價值：加速與普及
核心概念
NCLC 的核心價值在於：
  • 加速：大幅縮短應用程式的開發週期，快速回應業務需求。
  • 普及（民主化）：降低技術門檻，讓更多非專業開發人員也能參與創建數位解決方案。
(課程規劃書 L121)
#5
★★★
NCLC 的目標：提高生產力、降低成本、促進創新
核心概念
透過加速開發和賦能更多人，NCLC 旨在提高組織的整體生產力，減少對昂貴開發資源的依賴以降低成本，並鼓勵更多業務驅動的創新和實驗。(課程規劃書 L121)
#6
★★★★
主要特性：視覺化開發介面 (Visual Development Interface)
核心概念
這是 NCLC 平台的核心特徵。使用者透過圖形化使用者介面 (GUI - Graphical User Interface)，以拖放元件、設定屬性、連接流程等方式來建構應用程式，而非編寫程式碼。(課程規劃書 L121)
#7
★★★
主要特性：預構建元件與模板
核心概念
NCLC 平台通常提供豐富的預先構建好的功能模組（如表單、按鈕、圖表、資料連接器）和應用程式模板（如客戶關係管理模板、專案管理模板）。使用者可以直接選用並組合這些元件，快速搭建應用。(課程規劃書 L121)
#8
★★★
主要特性：模型驅動開發 (Model-Driven Development)
核心概念
許多 NCLC 平台採用模型驅動的方法，使用者定義資料模型、業務邏輯模型和使用者介面模型，平台根據這些模型自動生成應用程式的程式碼或執行環境。
#9
★★★★
目標使用者：公民開發者 (Citizen Developer)
核心概念
公民開發者是指具備業務知識但缺乏傳統程式設計技能的員工，他們利用 NCLC 平台來創建滿足其部門或團隊特定需求的應用程式。賦能公民開發者是 NCLC 的主要目標之一。(課程規劃書 L121)
#10
★★★
目標使用者：業務分析師與產品經理
核心概念
這些角色深入了解業務需求和流程，NCLC 平台使他們能夠更直接地將需求轉化為可運行的應用原型或解決方案，減少溝通成本和開發延遲。
#11
★★★
目標使用者：專業開發者 (Professional Developer)
核心概念
對於Low-Code平台而言，專業開發者也是重要使用者。他們可以利用平台快速完成標準化、重複性的開發工作，將精力集中在需要編寫程式碼的複雜功能、系統整合或效能優化上，從而提高整體開發效率。
#12
★★★★
與傳統開發在速度與成本上的差異
核心概念
相較於需要從頭編寫大量程式碼的傳統應用程式開發，NCLC開發速度更快，通常能節省數週甚至數月的時間。人力成本也相對較低，因為對專業開發者的依賴減少。(課程規劃書 L121)
#13
★★★★
與傳統開發在彈性與控制力上的差異
核心概念
傳統開發提供最大的彈性和控制力，開發者可以完全掌控應用程式的各個方面。NCLC 平台則犧牲了部分彈性以換取速度和易用性，使用者通常受限於平台提供的功能和架構。(課程規劃書 L121)
#14
★★★
與傳統開發在維護與更新上的差異
核心概念
NCLC 平台的底層架構和安全性更新通常由平台供應商負責，減輕了使用者的維護負擔。但應用程式本身的邏輯維護和功能更新仍需使用者進行，且可能受平台更新的影響。傳統開發則需要開發團隊自行負責所有層面的維護和更新。
