適用情境與選擇考量
#1
★★★★★
優勢：加速應用開發速度
核心概念
No-Code/Low-Code (NCLC) 平台提供預先構建的模組、模板和視覺化介面，讓使用者能透過拖放 (Drag-and-Drop) 或簡單配置的方式快速建構應用程式。大幅減少了傳統編寫程式碼所需的時間，從而加快產品上市時間 (Time-to-Market)。(課程規劃書 L121)
#2
★★★★
優勢：降低開發成本
核心概念
由於減少了對專業程式設計師的依賴以及縮短了開發週期，使用 NCLC 平台通常能降低應用程式開發的人力成本和總體擁有成本 (TCO - Total Cost of Ownership)。尤其對於資源有限的新創公司或中小型企業更具吸引力。(課程規劃書 L121)
#3
★★★★★
優勢：賦能非技術人員 (公民開發者)
核心概念
NCLC 平台最重要的優勢之一是降低了應用開發的技術門檻。讓不具備深厚程式設計背景的業務人員、分析師或設計師（常被稱為公民開發者 - Citizen Developer）也能參與應用的創建和修改，更貼近業務需求。(課程規劃書 L121)
#4
★★★★
優勢：快速原型設計 (Rapid Prototyping) 與概念驗證 (PoC)
核心概念
利用 NCLC 平台可以非常快速地搭建出應用程式的原型，用於展示核心功能、收集使用者回饋、驗證商業想法或技術可行性 (PoC - Proof of Concept)。這有助於在早期階段降低開發風險並加速決策過程。(課程規劃書 L121)
#5
★★★
優勢：促進創新與實驗
核心概念
由於開發速度快、成本低，NCLC 平台鼓勵團隊進行更多的嘗試和實驗。可以更容易地測試新想法、新功能或整合新的AI服務（例如透過 API 串接 GenAI），降低創新的門檻。
#6
★★★★★
限制：功能與客製化彈性受限
核心概念
這是 NCLC 最主要的限制之一。平台提供的功能模組和元件是預先定義好的，對於非常獨特、複雜或需要高度客製化的功能，平台可能無法支援或難以實現。使用者必須在平台的框架內工作。(課程規劃書 L121; 公部門手冊 P80)
#7
★★★
限制：特定業務邏輯的實現困難
核心概念
對於涉及複雜計算、特殊演算法或獨特業務規則的流程，NCLC 平台的標準功能可能難以直接實現。Low-Code 平台雖然允許部分程式碼編寫，但仍可能受到平台架構的限制。
#8
★★★★
限制：效能與可擴展性 (Scalability) 問題
核心概念
NCLC 平台生成的應用程式可能在效能上不如傳統手寫程式碼。當使用者數量、資料量或交易量大幅增加時，應用程式的效能可能下降，擴展性可能受限於平台的架構和資源限制。(課程規劃書 L121)
#9
★★★★
限制：供應商鎖定 (Vendor Lock-in)
核心概念
一旦在特定 NCLC 平台上開發應用程式，通常很難將其遷移到其他平台或環境。組織可能會高度依賴該平台供應商，若供應商更改服務條款、漲價或停止服務，可能對組織造成影響。(課程規劃書 L121)
#10
★★★
限制：與現有系統整合的複雜性
核心概念
將 NCLC 平台開發的應用與組織現有的複雜系統（如ERP、CRM、舊有系統）進行整合可能會遇到挑戰。雖然許多平台提供 API 接口，但整合的深度和彈性可能不如傳統開發方式。
#11
★★★★
限制：安全性與資料治理疑慮
核心概念
應用程式和資料儲存在第三方平台上，引發資料安全和主權的擔憂。組織需要仔細審查平台的安全措施、認證和合規性（如 GDPR、ISO 27001）。公民開發者可能缺乏足夠的安全意識，需要加強管理和培訓。(課程規劃書 L121)
#12
★★★
限制：合規性與稽核挑戰
核心概念
對於需要嚴格遵守特定行業法規（如金融、醫療）的應用，需要確保 NCLC 平台及其生成的應用符合相關的合規要求。此外，應用程式的變更追蹤和稽核記錄可能不如傳統開發流程完整，增加合規難度。
#13
★★★★
No-Code/Low-Code 的適用情境
核心概念
NCLC 平台特別適用於：
  • 內部工具與簡單流程自動化：如費用報銷、請假申請、簡單的資料管理應用。
  • 快速原型設計與MVP (Minimum Viable Product)：快速驗證市場或產品概念。
  • 簡單的客戶入口網站或登陸頁面。
  • 需要賦能業務部門自行解決簡單數位化需求的場景。
(課程規劃書 L121)
#14
★★★
選擇No-Code/Low-Code平台時的評估標準
核心概念
選擇平台時應考量：
  • 功能契合度：是否提供所需的核心功能和元件？
  • 易用性：介面是否直觀？學習曲線如何？
  • 整合能力：是否能與現有系統或第三方服務（如AI API）整合？
  • 客製化彈性：允許多少程度的自訂？（No-Code vs Low-Code）
  • 成本與定價模式。
  • 安全性與合規性。
  • 供應商支援與社群活躍度。
(課程規劃書 L121)
#15
★★★
優勢：提升IT與業務部門的協作
核心概念
NCLC 平台提供視覺化的開發環境，讓業務人員和IT人員能在共同的基礎上溝通和協作。業務人員可以更容易地表達需求並參與設計，IT人員則可以專注於更複雜的技術整合與治理。
